# 01 Product Overview

## What The System Does

Recurvos is a billing and subscription management platform. It lets subscriber businesses:

- maintain billing profiles for one or more companies
- manage customers
- define products and plans
- create manual invoices
- run recurring subscription billing
- track payments, refunds, and credit notes
- send invoices and reminders by email and optionally WhatsApp

It also lets the Recurvos platform owner:

- manage SaaS packages and feature limits
- assign packages to subscriber companies
- bill subscribers for package access
- manage platform settings, email, gateway, WhatsApp, feedback, and audits

## Target Users

- Subscriber business owners and admins using Recurvos for their own billing operations
- Platform owner/admin users running the Recurvos SaaS
- End customers receiving invoices or submitting payment confirmations

## Primary User Goals

- Set up a legally usable billing profile
- Create customers, products, and plans
- Start subscriptions and generate invoices on schedule
- Collect payment through links or manual confirmation
- Review payment confirmations and maintain payment history
- Keep access active by paying the Recurvos package invoice

## Core Business Flows

## 1. Subscriber onboarding

- User registers with package selection
- Account requires email verification
- A subscriber company is created
- Platform package invoice is generated for the selected package
- Package status starts as pending verification or pending payment depending on stage

## 2. Billing setup

- Subscriber configures company identity and invoice numbering
- Billing readiness checks ensure required issuer fields exist before invoice/subscription actions

## 3. Catalog setup

- Subscriber creates products and product plans
- Plans can be one-time or recurring
- Default plan logic exists per product

## 4. Subscription billing

- Subscriber creates subscriptions from plans
- Due invoices are generated by Hangfire
- Auto-send and reminder behavior depend on settings and package features

## 5. Payment collection

- Subscriber can record manual payments
- Subscriber can generate Billplz payment links when enabled
- Customer can submit a public payment confirmation with proof upload
- Staff reviews and approves or rejects pending confirmations

## 6. Platform package billing

- Platform owner assigns packages to subscriber companies
- Platform generates subscriber package invoices
- Upgrades are prorated within the current cycle
- Grace period and past-due logic affect feature access

## Business Value

- Consolidates invoice, subscription, and payment operations for SME-style billing
- Adds package-controlled SaaS monetization for the platform itself
- Supports Malaysian billing practices such as `MYR` currency and SST-style tax settings
- Provides operational visibility with audit logs, exports, reminders, and platform admin tools

## Current Scope

- Implemented now:
  - multi-tenant billing
  - package-based entitlement model
  - invoice PDFs and receipts
  - payment tracking
  - public payment confirmations
  - platform package billing
  - reminders, feedback, audit, exports
- Likely future scope:
  - richer reconciliation and settlement workflows
  - more payment gateways
  - broader dispute handling
  - more advanced finance/accounting behavior

Items in the second group should be treated as future-facing unless current code proves otherwise.
