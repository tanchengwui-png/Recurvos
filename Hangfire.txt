HANGFIRE / DUE INVOICE CHECK

Use this when you want to manually test the due-invoice job on staging.

1. SSH into the staging server
```bash
ssh server@115.187.22.70 -p 6262
```

2. Go to the staging repo
```bash
cd /srv/recurvos/staging/repo
```

3. Make sure the API container is running
```bash
docker compose -f docker-compose.staging.yml up -d api
docker compose -f docker-compose.staging.yml ps
```

4. Set the API base URL
```bash
API="http://localhost:7002"
```

5. Log in and save the access token into a variable
```bash
TOKEN=$(curl -s "$API/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"owner@recurvo.com","password":"Passw0rd!"}' \
  | sed -n 's/.*"accessToken":"\([^"]*\)".*/\1/p')
```

6. Check the token
```bash
echo "$TOKEN"
```

If it is empty, stop. Login failed.

7. Check how many due invoices exist before running the job
```bash
curl -s "$API/api/subscriptions/due-invoices/count" \
  -H "Authorization: Bearer $TOKEN"
```

8. Trigger the due-invoice job manually
```bash
curl -s -X POST "$API/api/subscriptions/run-due-invoices" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json"
```

9. Check the due invoice count again
```bash
curl -s "$API/api/subscriptions/due-invoices/count" \
  -H "Authorization: Bearer $TOKEN"
```

10. Optional: generate invoice for one subscription only

Get subscriptions:
```bash
curl -s "$API/api/subscriptions" \
  -H "Authorization: Bearer $TOKEN"
```

Copy one subscription id, then run:
```bash
curl -s -X POST "$API/api/subscriptions/<SUBSCRIPTION_ID>/generate-invoice" \
  -H "Authorization: Bearer $TOKEN"
```

QUICK SUMMARY

- Step 7 = check before
- Step 8 = run job
- Step 9 = check after
- If the number changes, the job ran
